## 024A-16 — Fix sync de grupos FB (extensión→PHP) + verificación de conexión
- **Que:** Corregido bug crítico de sync: la extensión enviaba campos en español/snake_case (`fb_group_id`, `nombre`, `tipo`, `cantidad_miembros`, `imagen_url`, `fuente`), pero el PHP esperaba inglés/camelCase (`id`, `name`, `type`, `memberCount`, `imageUrl`, `source`). Resultado: todos los grupos se saltaban con `errores++` y 0 se insertaban. Fix: PHP ahora acepta ambos formatos con fallback (`$grupo['id'] ?? $grupo['fb_group_id'] ?? ''`). También: nuevo endpoint `GET /grupos-fb/ping` para verificar conexión y token desde la extensión. Fix de URL: el dashboard WP mostraba la URL completa con `/grupos-fb`, pero la extensión añade `/grupos-fb/sync` a la base → doble path → 404. Ahora muestra solo la URL base.
- **Archivos:** `GruposFbRepository.php`, `GruposFbApiController.php`, `useSeccionConfigGruposFb.ts`, `SeccionConfigGruposFb.tsx`, `lecciones-aprendidas.md`
- **Extensión (sin git):** `background.ts` (función `pingApi()` + handler `PING_API`), `types.d.ts` (tipo `PING_API`), `dashboard.html` (botón "Verificar conexión"), `dashboard.ts` (handler click ping)
- **Gotchas:** El bug era silencioso — sync retornaba HTTP 200 con `{insertados:0, errores:N}` pero el éxito HTTP enmascaraba el fallo. El URL mismatch causaba 404 si el usuario copiaba la URL del dashboard en la extensión sin editar.
- **Sentinel:** No aplica nuevas reglas.
