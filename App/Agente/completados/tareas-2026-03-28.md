## 283A-1 — Panel actividad: heatmap estira al 100% del contenedor
- **Que:** El heatmap del panel de actividad no llenaba el ancho del contenedor cuando estaba en 1 columna o en contenedores anchos, dejando un espacio vacío enorme a la derecha. El problema era que `max-width` fijo (14/10/16px) en `.mapaCalorSemana` y `.mapaCalorCelda` impedía que las celdas crecieran más allá de esos límites, incluso cuando el flexbox tenía espacio sobrante. Este bug se reportó 5 veces antes con fixes que solo atacaban el interior del grid (overflow, min-width, aspect-ratio) sin resolver la raíz.
- **Archivos:**
  - `App/React/hooks/shared/useMapaCalor.ts` — Nuevo cálculo `tamanoCeldaReal`: mide el contenedor, resta márgenes, divide entre las semanas visibles para obtener el px exacto que llena 100%. También se usa para los anchos de etiquetas de meses (alineación correcta).
  - `App/React/components/shared/MapaCalor.tsx` — Inyecta `--mapa-calor-tamano-celda` como CSS variable en el contenedor raíz. Eliminadas clases de tamaño (`mapaCalorCelda--pequeno`, `--grande`, `mapaCalorContenedor--celdaPequeno`, `--celdaGrande`) que ya no son necesarias.
  - `App/React/styles/dashboard/shared/mapaCalor.css` — `max-width` de `.mapaCalorSemana` y `.mapaCalorCelda` ahora usan `var(--mapa-calor-tamano-celda, 14px)`. Eliminadas las 4 reglas de override por tamaño.
- **Gotchas:** 
  - El cálculo tiene un máximo de 30px y mínimo de 6px para evitar celdas absurdamente grandes (periodos cortos en containers anchos) o invisibles.
  - Las etiquetas de meses usan el mismo `tamanoCeldaReal + GAP_CELDAS` para mantener alineación perfecta con el grid.
  - El hook de hábitos (`useMapaCalorHabito`) NO se modificó — es un sistema separado.
- **Sentinel:** No aplica — fue un fix de diseño CSS/JS, no un patrón detectable estáticamente.
