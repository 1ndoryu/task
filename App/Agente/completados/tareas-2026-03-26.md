## 263A-1 — Fix cuadros panelActividadMapa overflow (3 tamaños configurables)
- **Que:** Corregido el overflow de cuadros del mapa de calor reportado 5 veces. Raíz: `TAMANO_CELDA` en JS no coincidía con `max-width` del CSS (JS: {10,12,14} vs CSS: {10px,14px,16px}), causando que el auto-cálculo generara más semanas de las que cabían. También `.mapaCalorSemana` tenía `max-width: 22px` hardcodeado sin adaptar al tamaño de celda.
- **Archivos:** `useMapaCalor.ts`, `MapaCalor.tsx`, `mapaCalor.css`, `mapaCalorHabito.css`, `PanelGruposFb.tsx` (fix import no usado)
- **Gotchas:** El desajuste entre JS y CSS existía desde siempre. Las 4 correcciones previas solo agregaban `overflow:hidden` o `min-width:0` a contenedores sin atacar la causa raíz: los valores de tamaño discordantes entre JS y CSS.
- **Sentinel:** No aplica — los tamaños son específicos de este componente.

## 263A-2 — Fix subhábitos fantasma que reaparecen + no se borran
- **Que:** Los subhábitos virtuales (IDs negativos) en el panel de ejecución no se podían eliminar porque el botón llamaba a `eliminarTarea` del tareasStore (donde no existen) → falla silenciosa → "reaparecen". Fix: interceptor `manejarEliminarTareaHabito` que rutea a `eliminarSubHabito` del habitosStore. También agregado `onRehydrate` al persist para limpiar subhábitos con nombre vacío de localStorage.
- **Archivos:** `useHabitosComoTareas.ts`, `useDashboardCompleto.ts`, `generadoresPropsPanel.ts`, `habitosStore.ts`
- **Gotchas:** El patrón de interception ya existía para toggle (`manejarToggleTareaHabito`) pero faltaba para delete. Los subhábitos fantasma (nombre vacío) se crearon antes de que se añadiera la validación y persistieron en localStorage.
- **Sentinel:** No aplica.

## 263A-3 — Duplicar panel notas (reemplaza "Abrir en ventana")
- **Que:** El menú + del panel Notas ahora ofrece "Duplicar panel" en lugar de "Abrir en ventana". Al duplicar, se crea una nueva instancia del scratchpad (e.g., `scratchpad-1`) en la misma columna, justo debajo del original. Los paneles duplicados tienen un botón X para cerrarse. Sistema genérico: `obtenerPanelOBase()` resuelve IDs con sufijo numérico al panel base del registro, `crearDuplicadoPanel()` y `eliminarPanelDuplicado()` son funciones puras en `layoutLogica.ts`.
- **Archivos:** `registroPaneles.ts`, `useConfiguracionLayout.ts`, `layoutLogica.ts`, `DashboardGrid.tsx`, `PanelScratchpad.tsx`
- **Gotchas:** IDs como `deficit-calorico` usan guión, así que el regex `^(.+)-(\d+)$` resuelve correctamente solo si la parte base está registrada. La migración de layout no elimina IDs desconocidos, así que las instancias duplicadas sobreviven recargas. Las funciones de duplicación se extrajeron a `layoutLogica.ts` para no exceder el límite de 300 líneas del hook.
- **Sentinel:** No aplica.
