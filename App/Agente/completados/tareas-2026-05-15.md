## 115A-5 — Chatbot: visión e imágenes/audios WhatsApp
- **Que:** Implementado soporte completo de media en el flujo WhatsApp.
  - `WacliService::descargarMedia()`: descarga media via `wacli media download --direct-path ... --media-key ... --output ...` a /tmp.
  - `LLMProviderService::transcribirAudio()`: transcripción con Groq Whisper (multipart form, `whisper-large-v3`). Idioma ES.
  - `LLMProviderService::validarMensajes()`: ahora acepta content array (multimodal) además de string.
  - `AgentChatProcessor::procesar()`: param opcional `?array $media`. Si es imagen, usa `meta-llama/llama-4-scout-17b-16e-instruct` con content array multimodal. Para audio, el transcript ya llega como texto normal.
  - `WhatsAppWebhookService::procesarEvento()`: detección formato nuevo actualizada para cubrir eventos con solo Media. Extrae DirectPath/MediaKey/Type, descarga, procesa según tipo, limpia /tmp. Error de media → mensaje de aviso al usuario si no hay texto alternativo.
- **Archivos:** WacliService.php, LLMProviderService.php, AgentChatProcessor.php, WhatsAppWebhookService.php
- **Gotchas:**
  - El comando exacto `wacli media download` puede variar por versión — verificar con `wacli media --help` si falla.
  - whisper-large-v3 agregado al array PROVIDERS['groq']['models'] solo por consistencia; no se llama via enviarChat().
  - file_get_contents en imagen: si falla devuelve "", base64_encode("") = "" → agente degrada a texto, no crash.
- **Sentinel:** No aplica
- **GLORY:** No aplica
