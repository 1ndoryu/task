## Explicación OpenCode como LLM

OpenCode es un agente de código autónomo (tool-use loops, 20-60s por ciclo). NO es viable como LLM de chat porque:
1. Latencia: cada "respuesta" implica múltiples tool-calls encadenadas
2. Diseñado para tareas de código (archivos, builds, tests), no conversación
3. No expone endpoint de chat directo

La alternativa correcta para "modo pensamiento" es `deepseek-reasoner` ya agregado en LLMProviderService. Para activarlo:
```php
update_user_meta($userId, 'glory_chatbot_proveedor', 'deepseek');
update_user_meta($userId, 'glory_chatbot_modelo', 'deepseek-reasoner');
```
